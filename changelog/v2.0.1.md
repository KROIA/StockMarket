# v2.0.1 ŌĆö In Development

Plugin system UI, runtime data streaming, management screen improvements, and traded volume charting.

## API

| Tag | Change |
|---:|---|
| ![feature] | <details><summary>Plugin developer API simplified to two classes</summary><br>Plugin developers only need to implement `ServerPlugin` (server logic) and optionally `PluginGuiElement` (custom UI). No more `ClientPlugin` subclassing required. GUI element factories registered in `clientSetup()` for dedicated server safety.</details> |
| ![feature] | `PluginGuiElement.needsCustomScreen()` ŌĆö plugins choose between inline settings or a dedicated full-screen view |
| ![feature] | <details><summary>Type-safe generic plugin API with StreamCodec</summary><br>`ServerPlugin<TSettings, TRuntimeData>` and `PluginGuiElement<TSettings, TRuntimeData>` use typed records with `StreamCodec<ByteBuf, T>` for custom settings and runtime data. Framework handles byte[] encoding internally ŌĆö plugin developers never touch raw bytes.</details> |
| ![feature] | `CandlestickChart.getCanvasBounds()` ŌĆö exposed canvas bounds for external overlay rendering via coordinate transforms |
| ![feature] | <details><summary>Generic overlay interface for `CandlestickChart` and `OrderbookVolumeHistogram`</summary><br>Both chart widgets now expose `Overlay` functional interfaces with `addOverlay()`/`removeOverlay()` registration. Overlays render inside scissor-clipped canvas bounds with access to coordinate conversion methods (`toCanvasSpaceX`, `toCanvasSpaceY`, `getCanvasBounds`). Any plugin can register callbacks on either widget.</details> |
| ![feature] | `PluginGuiElement.setCandlestickChart()` and `setOrderbookVolumeHistogram()` ŌĆö hooks for plugins to register overlays on shared chart widgets from any screen context |
| ![feature] | <details><summary>Traded volume tracking across the full data pipeline</summary><br>`MatchingEngine` accumulates matched trade volume per update cycle. `ServerMarket` stores cumulative volume per candle period. `MarketPriceStruct`, `PriceHistoryData.Candle`, SQL schema, and `MarketPriceStream` all carry the new `tradedVolume` float field. Candle merge sums volumes across aggregated timeframes.</details> |
| ![feature] | `ISyncServerMarket.getCurrentCandleTradedVolume()` ŌĆö exposes live candle volume for network streaming |

## Internal

| Tag | Change |
|---:|---|
| ![feature] | <details><summary>Plugin runtime data streaming via `PluginRuntimeDataStream`</summary><br>Server-to-client stream following MarketPriceStream pattern. Per-plugin subscription with configurable update interval. TargetPriceBot streams target prices at 200ms intervals.</details> |
| ![feature] | `PluginListRequest` ŌĆö ARRS request to fetch plugin list with metadata and subscriptions |
| ![feature] | `PluginSettingsRequest` ŌĆö ARRS request to update generic plugin settings (enable, name, description, logger) |
| ![feature] | `PluginReorderRequest` ŌĆö ARRS request to reorder plugin execution order |
| ![feature] | `PluginCustomSettingsRequest` ŌĆö ARRS request to update plugin-specific custom settings |
| ![feature] | `PluginSubscriptionRequest` ŌĆö ARRS request to subscribe/unsubscribe a plugin to/from a market |
| ![feature] | <details><summary>Plugin persistence via NBT save/load</summary><br>Plugin instances, execution order, market subscriptions, custom settings, and plugin-specific data survive server restarts. Fresh worlds fall back to default plugin creation.</details> |
| ![feature] | `PluginSyncData` ŌĆö network-transferable plugin state bundle with `StreamCodec`, now includes custom settings |
| ![improvement] | `PluginSyncData.fromServerPlugin()` ŌĆö factory method to build sync data from a server plugin instance |
| ![improvement] | `ServerPluginManager` uses `LinkedHashMap` for deterministic plugin execution order |
| ![bugfix] | `GenericPluginData` private constructor not assigning `enabled` field ŌĆö deserialized plugins always showed as disabled |
| ![bugfix] | `AsyncMarketManager.createMarketAsync()` sent `FunctionType.MarketExists` instead of `FunctionType.CreateMarket` |
| ![bugfix] | `AsyncMarketManager.deleteMarketAsync()` sent `FunctionType.MarketExists` instead of `FunctionType.DeleteMarket` |
| ![improvement] | VolatilityPlugin refactored to per-market independent random walk generators (previously shared single generator) |
| ![improvement] | TargetPriceBot and VolatilityPlugin save/load per-market runtime state (PID controller, target prices, random walk generators) |
| ![improvement] | <details><summary>TargetPriceBotGuiElement refactored to generic overlay pattern</summary><br>Stores all market target prices in a map instead of tracking a single selected market. Registers overlays via `CandlestickChart.Overlay` interface on both its own chart (dedicated screen) and the shared chart (management screen). Old manual coordinate-offset rendering in `render()` replaced by overlay callback.</details> |
| ![improvement] | Overlay cleanup in `PluginManagementScreen.applyFilter()` ŌĆö removes stale overlays and stops data streams when plugin entry widgets are rebuilt |
| ![feature] | `DatabaseManager.migrateSchema()` ŌĆö automatically adds new SQL columns to existing databases on startup (traded_volume migration) |
| ![hardening] | Plugin registration split into server-safe static init and client-only `clientSetup()` to prevent dedicated server crash from loading GUI classes |
