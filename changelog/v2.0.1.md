# v2.0.1 ‚Äî In Development

Plugin system UI, runtime data streaming, and management screen improvements.

## API

| Tag | Change |
|---:|---|
| ![feature] | <details><summary>Plugin developer API simplified to two classes</summary><br>Plugin developers only need to implement `ServerPlugin` (server logic) and optionally `PluginGuiElement` (custom UI). No more `ClientPlugin` subclassing required. GUI element factories registered in `clientSetup()` for dedicated server safety.</details> |
| ![feature] | `PluginGuiElement.needsCustomScreen()` ‚Äî plugins choose between inline settings or a dedicated full-screen view |
| ![feature] | <details><summary>Type-safe generic plugin API with StreamCodec</summary><br>`ServerPlugin<TSettings, TRuntimeData>` and `PluginGuiElement<TSettings, TRuntimeData>` use typed records with `StreamCodec<ByteBuf, T>` for custom settings and runtime data. Framework handles byte[] encoding internally ‚Äî plugin developers never touch raw bytes.</details> |
| ![feature] | `CandlestickChart.getCanvasBounds()` ‚Äî exposed canvas bounds for external overlay rendering via coordinate transforms |

| ![feature] | VolumeDistributionPlugin inline settings UI ‚Äî configurable volume scale and convergence speed with Apply button |

## Internal

| Tag | Change |
|---:|---|
| ![feature] | <details><summary>Plugin runtime data streaming via `PluginRuntimeDataStream`</summary><br>Server-to-client stream following MarketPriceStream pattern. Per-plugin subscription with configurable update interval. TargetPriceBot streams target prices at 200ms intervals.</details> |
| ![feature] | `PluginListRequest` ‚Äî ARRS request to fetch plugin list with metadata and subscriptions |
| ![feature] | `PluginSettingsRequest` ‚Äî ARRS request to update generic plugin settings (enable, name, description, logger) |
| ![feature] | `PluginReorderRequest` ‚Äî ARRS request to reorder plugin execution order |
| ![feature] | `PluginCustomSettingsRequest` ‚Äî ARRS request to update plugin-specific custom settings |
| ![feature] | `PluginSubscriptionRequest` ‚Äî ARRS request to subscribe/unsubscribe a plugin to/from a market |
| ![feature] | <details><summary>Plugin persistence via NBT save/load</summary><br>Plugin instances, execution order, market subscriptions, custom settings, and plugin-specific data survive server restarts. Fresh worlds fall back to default plugin creation.</details> |
| ![feature] | `PluginSyncData` ‚Äî network-transferable plugin state bundle with `StreamCodec`, now includes custom settings |
| ![improvement] | `PluginSyncData.fromServerPlugin()` ‚Äî factory method to build sync data from a server plugin instance |
| ![improvement] | `ServerPluginManager` uses `LinkedHashMap` for deterministic plugin execution order |
| ![bugfix] | `GenericPluginData` private constructor not assigning `enabled` field ‚Äî deserialized plugins always showed as disabled |
| ![bugfix] | `AsyncMarketManager.createMarketAsync()` sent `FunctionType.MarketExists` instead of `FunctionType.CreateMarket` |
| ![bugfix] | `AsyncMarketManager.deleteMarketAsync()` sent `FunctionType.MarketExists` instead of `FunctionType.DeleteMarket` |
| ![improvement] | VolatilityPlugin refactored to per-market independent random walk generators (previously shared single generator) |
| ![improvement] | TargetPriceBot and VolatilityPlugin save/load per-market runtime state (PID controller, target prices, random walk generators) |
| ![hardening] | Plugin registration split into server-safe static init and client-only `clientSetup()` to prevent dedicated server crash from loading GUI classes |
