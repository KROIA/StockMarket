# v2.0.1 — In Development

Plugin system UI, runtime data streaming, management screen improvements, and traded volume charting.

## API

| Tag | Change |
|---:|---|
| ![feature] | <details><summary>Plugin developer API simplified to two classes</summary><br>Plugin developers only need to implement `ServerPlugin` (server logic) and optionally `PluginGuiElement` (custom UI). No more `ClientPlugin` subclassing required. GUI element factories registered in `clientSetup()` for dedicated server safety.</details> |
| ![feature] | `PluginGuiElement.needsCustomScreen()` — plugins choose between inline settings or a dedicated full-screen view |
| ![feature] | <details><summary>Type-safe generic plugin API with StreamCodec</summary><br>`ServerPlugin<TSettings, TRuntimeData>` and `PluginGuiElement<TSettings, TRuntimeData>` use typed records with `StreamCodec<ByteBuf, T>` for custom settings and runtime data. Framework handles byte[] encoding internally — plugin developers never touch raw bytes.</details> |
| ![feature] | `CandlestickChart.getCanvasBounds()` — exposed canvas bounds for external overlay rendering via coordinate transforms |
| ![feature] | <details><summary>Generic overlay interface for `CandlestickChart` and `OrderbookVolumeHistogram`</summary><br>Both chart widgets now expose `Overlay` functional interfaces with `addOverlay()`/`removeOverlay()` registration. Overlays render inside scissor-clipped canvas bounds with access to coordinate conversion methods (`toCanvasSpaceX`, `toCanvasSpaceY`, `getCanvasBounds`). Any plugin can register callbacks on either widget.</details> |
| ![feature] | <details><summary>`CandlestickChart.InteractiveOverlay` — overlays that receive mouse events</summary><br>Extends `Overlay` with `mouseClicked`, `mouseDragged`, `mouseReleased` methods. Chart forwards mouse events to interactive overlays before processing its own input. If an overlay consumes an event, chart panning is suppressed.</details> |
| ![feature] | `StockMarketGuiElement.drawXMark()` — shared utility for drawing close/cancel X marks across UI elements |
| ![feature] | `PluginGuiElement.setCandlestickChart()` and `setOrderbookVolumeHistogram()` — hooks for plugins to register overlays on shared chart widgets from any screen context |
| ![feature] | <details><summary>Traded volume tracking across the full data pipeline</summary><br>`MatchingEngine` accumulates matched trade volume per update cycle. `ServerMarket` stores cumulative volume per candle period. `MarketPriceStruct`, `PriceHistoryData.Candle`, SQL schema, and `MarketPriceStream` all carry the new `tradedVolume` float field. Candle merge sums volumes across aggregated timeframes.</details> |
| ![feature] | `ISyncServerMarket.getCurrentCandleTradedVolume()` — exposes live candle volume for network streaming |
| ![feature] | <details><summary>Market preset system with JSON configuration</summary><br>`MarketPresetManager` loads item presets from `config/stockmarket/market_presets/` (one JSON file per category). Each preset defines `defaultPrice` and `naturalAbundance`. On first startup, generates defaults for 5 categories (CommonBlock, Ore, Food, Gardening, AnimalLoot) with ~76 items. Server admins can edit the JSON files to customize values.</details> |
| ![feature] | `MarketSettings.naturalAbundance` — per-market volume scale factor derived from item rarity |
| ![feature] | <details><summary>Plugin auto-subscribe on market creation</summary><br>Plugins gain `autoSubscribeNewMarkets` (boolean, default true) and `subscriptionOrder` (int, 0=earliest) fields. When a new market is created, `ServerPluginManager.autoSubscribeNewMarket()` subscribes all enabled opt-in plugins sorted by order. Fields persist via NBT and sync to client via codec.</details> |
| ![feature] | <details><summary>Player preferences API — per-player favorites and last market</summary><br>`PlayerPreferences` data class with `lastMarketID` and `favoriteMarketIDs` (stable add-order). Implements `ServerSaveable` (NBT) and `STREAM_CODEC` (network). Stored in `User`, persisted across server restarts. `ISyncServerMarketManager` and `IAsyncMarketManager` extended with `getPlayerPreferences()`/`updatePlayerPreferences()`. Client-side cache fetched on player join.</details> |
| ![feature] | `PlayerPreferencesGetRequest` and `PlayerPreferencesUpdateRequest` — ARRS requests for syncing player trading preferences with the master server |

| Tag | Change |
|---:|---|
| ![feature] | <details><summary>Plugin management screen with full CRUD controls</summary><br>Accessible from the ManagementScreen via "Manage Plugins" button. Shows all plugins with name, description, subscribed markets (as item icons), enable/disable checkbox, and up/down reorder buttons for execution order.</details> |
| ![feature] | Plugin overview widget in ManagementScreen showing enabled/total plugin count |
| ![feature] | `PluginDetailScreen` — dedicated screen for plugins that need custom full-screen UI |
| ![feature] | Market filter popup in PluginManagementScreen — ItemSelectionView for filtering plugins by subscribed market |
| ![feature] | Market creation screen — "+" button in ManagementScreen opens item selection to create new markets |
| ![feature] | TargetPriceBot custom screen with candlestick chart, market selector, and live target price overlay line |
| ![feature] | TargetPriceBot PID settings UI — editable P/I/D gains and rate with Apply button, synced to server via custom settings request |
| ![feature] | VolatilityPlugin inline settings UI — configurable volatility scale with Apply button |
| ![feature] | VolumeDistributionPlugin inline settings UI — configurable volume scale and convergence speed with Apply button |
| ![feature] | Subscribe/unsubscribe markets to plugins via "+" and "x" buttons in plugin management screen |
| ![feature] | <details><summary>PluginManagementScreen redesigned with candlestick chart and orderbook histogram</summary><br>Left side shows a candlestick chart with an orderbook volume histogram; right side shows the plugin list. Clicking any market item icon across any plugin selects that market for the chart — all icons for the same market highlight with a green overlay. Each market icon has a built-in close button (top-right corner) for unsubscribing.</details> |
| ![feature] | TargetPriceBot target price line now visible in PluginManagementScreen chart (not just the dedicated plugin screen) |
| ![feature] | VolumeDistribution target volume curve overlay on orderbook histogram — orange line showing the distribution shape the bot is targeting |
| ![feature] | <details><summary>ManagementScreen redesigned with TabElement</summary><br>Combined overview and market creation into a tabbed layout. "Overview" tab contains the existing chart, market list, and plugin controls. "Create Markets" tab provides category-based browsing with toggleable item selection (green overlay), existing-market detection (greyed out), and batch creation.</details> |
| ![feature] | <details><summary>CreateMarketTab — category-based batch market creation</summary><br>Category buttons load from `MarketPresetManager`. Item grid shows items from the selected category with search filtering. Items toggle with green overlay for multi-selection. Already-existing markets are greyed out. Selected items listed on the right with preset price/abundance. "Create All" button batch-creates selected markets.</details> |
| ![feature] | Plugin auto-subscribe checkbox and subscription order field in PluginManagementScreen entry widgets |
| ![feature] | <details><summary>TradeScreen redesigned with market switching and favorites</summary><br>Top-right panel shows a scrollable grid of all available markets as `MarketFavoriteButton` icons. Each icon has a 5-pointed star overlay (gold/gray) to toggle favorite status. Favorites pinned to top of grid, non-favorites sorted alphabetically. Selected market highlighted with green overlay and shown as a 24×24 icon. Last selected market restored on screen reopen via player preferences.</details> |
| ![feature] | `MarketFavoriteButton` — clickable item icon with pentagram star overlay for toggling favorites, press-and-release semantics on star area |
| ![feature] | `MarketSelectorPopup` — full-screen market selection popup using `ItemSelectionView` grid (unused after grid redesign, kept for future use) |
| ![feature] | <details><summary>Pending orders panel with real-time updates and cancel</summary><br>Bottom-left tab element in TradeScreen shows active orders streamed from the server via `ActiveOrdersStream`. Each entry displays item icon, BUY/SELL type, filled/total amount, price, and a cancel button. Orders sorted most-recent first. Cancel sends `CancelOrderRequest` with composite key matching.</details> |
| ![feature] | <details><summary>Draggable order markers on candlestick chart</summary><br>Pending limit orders appear as horizontal lines on the chart at their price level, colored green (buy) or red (sell). Each marker has a drag handle showing the remaining volume and a cancel [X] button. Drag the handle to move an order to a new price (cancels old, creates new with remaining volume). Markers occupy the right half of the chart.</details> |
| ![improvement] | Pending orders panel item icons are clickable to switch markets — updates chart, trading panel, and favorites bar selection |
| ![feature] | <details><summary>Plugin creation and deletion in PluginManagementScreen</summary><br>Admins can add new plugin instances from registered types via "Add Plugin" popup listing all available plugin types. Each plugin entry has a delete button ("x") with confirmation. Both operations require op level 2 and are backed by `PluginCreateRequest` and `PluginDeleteRequest` ARRS requests.</details> |
| ![feature] | <details><summary>Preset editor tab in ManagementScreen</summary><br>Third tab "Presets" allows admins to browse categories, search items, and edit default price/abundance values via inline text fields. Save button sends `PresetUpdateRequest` to the server which persists changes to JSON. Only changed presets are transmitted. Edit tracking via dirty map.</details> |
| ![feature] | <details><summary>Order history and transaction history tabs</summary><br>"My History" tab shows the player's own completed/cancelled order records from SQL, loaded on demand via `OrderHistoryRequest`. "Market Trades" tab shows all players' trades for the currently selected market via `TransactionHistoryRequest`, refreshed on market switch. Both display amount, price, and timestamp with real-value conversion.</details> |
| ![feature] | <details><summary>Traded volume bars in the candlestick chart</summary><br>Dedicated volume area below the candlestick candles (bottom 1/5 of chart). Bars show total executed trade volume per candle period with green/red coloring matching the candle direction. Volume data streams live from the server and persists in SQL.</details> |
