# v2.0.1 — 2026-05-21

Plugin system UI, runtime data streaming, management screen improvements, and traded volume charting.

## API

| Tag | Change |
|---:|---|
| ![feature] | <details><summary>Plugin developer API simplified to two classes</summary><br>Plugin developers only need to implement `ServerPlugin` (server logic) and optionally `PluginGuiElement` (custom UI). No more `ClientPlugin` subclassing required. GUI element factories registered in `clientSetup()` for dedicated server safety.</details> |
| ![feature] | `PluginGuiElement.needsCustomScreen()` — plugins choose between inline settings or a dedicated full-screen view |
| ![feature] | <details><summary>Type-safe generic plugin API with StreamCodec</summary><br>`ServerPlugin<TSettings, TRuntimeData>` and `PluginGuiElement<TSettings, TRuntimeData>` use typed records with `StreamCodec<ByteBuf, T>` for custom settings and runtime data. Framework handles byte[] encoding internally — plugin developers never touch raw bytes.</details> |
| ![feature] | `CandlestickChart.getCanvasBounds()` — exposed canvas bounds for external overlay rendering via coordinate transforms |
| ![feature] | <details><summary>Generic overlay interface for `CandlestickChart` and `OrderbookVolumeHistogram`</summary><br>Both chart widgets now expose `Overlay` functional interfaces with `addOverlay()`/`removeOverlay()` registration. Overlays render inside scissor-clipped canvas bounds with access to coordinate conversion methods (`toCanvasSpaceX`, `toCanvasSpaceY`, `getCanvasBounds`). Any plugin can register callbacks on either widget.</details> |
| ![feature] | <details><summary>`CandlestickChart.InteractiveOverlay` — overlays that receive mouse events</summary><br>Extends `Overlay` with `mouseClicked`, `mouseDragged`, `mouseReleased` methods. Chart forwards mouse events to interactive overlays before processing its own input. If an overlay consumes an event, chart panning is suppressed.</details> |
| ![feature] | `StockMarketGuiElement.drawXMark()` — shared utility for drawing close/cancel X marks across UI elements |
| ![feature] | `PluginGuiElement.setCandlestickChart()` and `setOrderbookVolumeHistogram()` — hooks for plugins to register overlays on shared chart widgets from any screen context |
| ![feature] | <details><summary>Traded volume tracking across the full data pipeline</summary><br>`MatchingEngine` accumulates matched trade volume per update cycle. `ServerMarket` stores cumulative volume per candle period. `MarketPriceStruct`, `PriceHistoryData.Candle`, SQL schema, and `MarketPriceStream` all carry the new `tradedVolume` float field. Candle merge sums volumes across aggregated timeframes.</details> |
| ![feature] | `ISyncServerMarket.getCurrentCandleTradedVolume()` — exposes live candle volume for network streaming |
| ![feature] | <details><summary>Market preset system with JSON configuration</summary><br>`MarketPresetManager` loads item presets from `config/stockmarket/market_presets/` (one JSON file per category). Each preset defines `defaultPrice` and `naturalAbundance`. On first startup, generates defaults for 5 categories (CommonBlock, Ore, Food, Gardening, AnimalLoot) with ~76 items. Server admins can edit the JSON files to customize values.</details> |
| ![feature] | `MarketSettings.naturalAbundance` — per-market volume scale factor derived from item rarity |
| ![feature] | <details><summary>Plugin auto-subscribe on market creation</summary><br>Plugins gain `autoSubscribeNewMarkets` (boolean, default true) and `subscriptionOrder` (int, 0=earliest) fields. When a new market is created, `ServerPluginManager.autoSubscribeNewMarket()` subscribes all enabled opt-in plugins sorted by order. Fields persist via NBT and sync to client via codec.</details> |
| ![feature] | <details><summary>Player preferences API — per-player favorites and last market</summary><br>`PlayerPreferences` data class with `lastMarketID` and `favoriteMarketIDs` (stable add-order). Implements `ServerSaveable` (NBT) and `STREAM_CODEC` (network). Stored in `User`, persisted across server restarts. `ISyncServerMarketManager` and `IAsyncMarketManager` extended with `getPlayerPreferences()`/`updatePlayerPreferences()`. Client-side cache fetched on player join.</details> |
| ![feature] | `PlayerPreferencesGetRequest` and `PlayerPreferencesUpdateRequest` — ARRS requests for syncing player trading preferences with the master server |
| ![feature] | <details><summary>Market presets support data-driven items (enchanted books, potions, tipped arrows)</summary><br>`MarketPreset` extended with optional data component field for items with NBT components. Component-aware identity via `getUniqueKey()`. Serialized via `RegistryOps` for correct data-driven registry resolution. `STREAM_CODEC` encodes components for network transport.</details> |
| ![feature] | <details><summary>Server-authoritative preset management via `AsyncPresetManager`</summary><br>Market presets now synced from the master server instead of loaded locally. `IAsyncPresetManager` interface with `getCategoriesAsync()`, `saveCategoryAsync()`, `deleteCategoryAsync()`, `renameCategoryAsync()`. Follows the `AsyncMarketManager` ARRS forwarding pattern with op level 2 permission checks.</details> |
| ![feature] | `IPriceDataProvider.getViewportKey()` — overrideable viewport cache key for chart providers with non-unique item IDs (used by cross-rate pairs) |
| ![feature] | `IPluginMarket.getNaturalAbundance()` — expose per-market item rarity factor to plugins |
| ![feature] | `IPluginMarket.getTargetPrice()` — read the current target price set by plugins this tick |
| ![feature] | `IPluginMarket.isMarketOpen()` — check whether a market is open for trading |
| ![feature] | `IPluginMarket.getCurrentCandleTradedVolume()` — read accumulated traded volume for the current candle |
| ![feature] | `IPluginOrderBook.getCapital(long, long)` — query total monetary depth in a backend price range |
| ![feature] | `IPluginOrderBook.getBuyOrders(long, long)` and `getSellOrders(long, long)` — filter orders by backend price range |
| ![improvement] | Auto-subscribe now works for disabled plugins — markets accumulate subscriptions and take effect when enabled |

## Inter-Market Trading

| Tag | Change |
|---:|---|
| ![feature] | <details><summary>Inter-market (item-to-item) trading system</summary><br>Players can trade items directly between markets without manually selling and buying. Two order types: market orders (immediate two-leg execution) and limit orders (bilateral matching with cross-rate enforcement). Money flows through the player's bank as an intermediary.</details> |
| ![feature] | <details><summary>Bilateral matching engine for cross-market limit orders</summary><br>Replaces the old sell-first-then-buy approach which had rate violations and over-selling bugs. Walks both orderbooks simultaneously — have-market buy-side DOWN, want-market sell-side UP. Money flows through a `transactionMoneyBalance` buffer on the order. Enforces cross-rate limit at every price level. Consumes both virtual depth and real player limit orders with counterparty bank operations.</details> |
| ![feature] | <details><summary>Cross-rate candlestick chart for pair trading</summary><br>CandlestickChart supports cross-rate mode showing synthetic OHLC candles derived from two markets. `CrossRateMarket` computes candles, tracks per-candle high/low extremes, and provides `IPriceDataProvider` interface. Live cross-rate updates each frame.</details> |
| ![feature] | <details><summary>Trade screen pair mode with pair selector</summary><br>Mode toggle switches between money-mode and pair-mode. `PairSelectorWidget` shows I WANT / I HAVE dropdowns for selecting the trading pair. Chart switches to cross-rate view. Order markers show inter-market limit orders at their rate limit price with cancel buttons.</details> |
| ![feature] | <details><summary>Inter-market trading panels</summary><br>Market exchange panel for instant item-to-item swaps. Limit exchange panel for placing cross-rate limit orders with rate input. Both panels show estimated yield, cost labels, and validate against player balances. Buy/Sell buttons with want-item quantities.</details> |
| ![feature] | `PlaceInterMarketOrderRequest` — ARRS request for placing inter-market orders with rate limit validation and item locking |
| ![feature] | `CancelInterMarketOrderRequest` — ARRS request for cancelling pending cross-market orders |
| ![feature] | `GetAvailablePairsRequest` — ARRS request for fetching tradeable market pairs |
| ![feature] | `InterMarketOrder` with `transactionMoneyBalance` field — dollar buffer persisted via NBT across partial fill ticks |
| ![feature] | Inter-market order history with `interMarketGroupID` linking both legs in the order record database |
| ![feature] | Player pair preferences — last selected have/want markets restored on screen reopen |
| ![feature] | <details><summary>87 unit tests for inter-market execution (InterMarketExecutorTestSuite)</summary><br>Covers basic bilateral fill, rate enforcement, stop conditions, depth shapes, real order interaction with counterparty bank ops, money buffer lifecycle, bot orders, multi-order concurrency, edge cases, balance conservation, and price movement.</details> |
| ![bugfix] | <details><summary>Bilateral walk no longer drains virtual orderbook on skipped orders</summary><br>Price updates now use `setCurrentMarketPriceNoRedistribute()` which avoids shifting the VirtualOrderbook's DynamicIndexedArray window. Only updates market price when fills actually occurred, using the last fill price instead of the walk-past position.</details> |
| ![bugfix] | Fix `haveLockedVolume` depleting before `wantTargetVolume` due to integer rounding — algorithm continues buying with remaining `transactionMoneyBalance` buffer |
| ![improvement] | <details><summary>Inter-market order markers: color, labels, and drag support</summary><br>Markers now colored green (buy) / red (sell) instead of always blue/cyan. Labels show pending buy volume with ≤ symbol. Markers are draggable to change the rate limit — cancels old order and places a new one at the dragged rate, preserving volume and direction.</details> |
| ![bugfix] | Fix order move preserving the wrong volume — buy direction now keeps the want-volume (buy target) constant and recalculates have-volume from the new rate |
| ![bugfix] | Fix order move volume drift from floating-point imprecision — uses integer arithmetic (`rawHaveVol = rawBuyVol * rawRate / SF`) matching the server's reconstruction formula |
| ![bugfix] | Fix cross-rate chart viewport sharing state with the underlying market — each pair now uses a unique `pair:have/want` viewport cache key |
| ![bugfix] | Fix `PairSelectorWidget` dropdown grid layout timing |
| ![bugfix] | <details><summary>Fix inter-market rounding truncation from cascading floor division</summary><br>Floor division (`funds * SF / price`) systematically under-counts by 1. Added `affordableVolume()` helper that checks if +1 is still affordable. Replaced all raw floor-division sites in `MatchingEngine` and `InterMarketExecutor`. Simplified `PlaceInterMarketOrderRequest` estimate to single division. Used `getOrCreateBank` for inter-market buy legs. Removed redundant dust deposit in `InterMarketExecutor`.</details> |
