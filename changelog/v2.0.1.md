# v2.0.1 — In Development

Plugin system UI, runtime data streaming, and management screen improvements.

## API

| Tag | Change |
|---:|---|
| ![feature] | <details><summary>Plugin developer API simplified to two classes</summary><br>Plugin developers only need to implement `ServerPlugin` (server logic) and optionally `PluginGuiElement` (custom UI). No more `ClientPlugin` subclassing required. GUI element factories registered in `clientSetup()` for dedicated server safety.</details> |
| ![feature] | `PluginGuiElement.needsCustomScreen()` — plugins choose between inline settings or a dedicated full-screen view |
| ![feature] | `ServerPlugin.provideRuntimeData()` — opt-in live data streaming from server plugin to client GUI element |
| ![feature] | <details><summary>Custom plugin settings API via opaque byte[] payloads</summary><br>`ServerPlugin.provideCustomSettings()` and `applyCustomSettings(byte[])` let each plugin define its own settings format. Settings are included in `PluginSyncData` and editable through `PluginGuiElement.sendCustomSettings()`.</details> |
| ![feature] | `CandlestickChart.getCanvasBounds()` — exposed canvas bounds for external overlay rendering via coordinate transforms |

## UI / UX

| Tag | Change |
|---:|---|
| ![feature] | <details><summary>Plugin management screen with full CRUD controls</summary><br>Accessible from the ManagementScreen via "Manage Plugins" button. Shows all plugins with name, description, subscribed markets (as item icons), enable/disable checkbox, and up/down reorder buttons for execution order.</details> |
| ![feature] | Plugin overview widget in ManagementScreen showing enabled/total plugin count |
| ![feature] | `PluginDetailScreen` — dedicated screen for plugins that need custom full-screen UI |
| ![feature] | Market filter popup in PluginManagementScreen — ItemSelectionView for filtering plugins by subscribed market |
| ![feature] | Market creation screen — "+" button in ManagementScreen opens item selection to create new markets |
| ![feature] | TargetPriceBot custom screen with candlestick chart, market selector, and live target price overlay line |
| ![feature] | TargetPriceBot PID settings UI — editable P/I/D gains and rate with Apply button, synced to server via custom settings request |
| ![feature] | VolatilityPlugin inline settings UI — configurable volatility scale with Apply button |
| ![feature] | VolumeDistributionPlugin inline settings UI — configurable volume scale and convergence speed with Apply button |
| ![feature] | Subscribe/unsubscribe markets to plugins via "+" and "x" buttons in plugin management screen |

## Internal

| Tag | Change |
|---:|---|
| ![feature] | <details><summary>Plugin runtime data streaming via `PluginRuntimeDataStream`</summary><br>Server-to-client stream following MarketPriceStream pattern. Per-plugin subscription with configurable update interval. TargetPriceBot streams target prices at 200ms intervals.</details> |
| ![feature] | `PluginListRequest` — ARRS request to fetch plugin list with metadata and subscriptions |
| ![feature] | `PluginSettingsRequest` — ARRS request to update generic plugin settings (enable, name, description, logger) |
| ![feature] | `PluginReorderRequest` — ARRS request to reorder plugin execution order |
| ![feature] | `PluginCustomSettingsRequest` — ARRS request to update plugin-specific custom settings (opaque byte[] payload) |
| ![feature] | `PluginSubscriptionRequest` — ARRS request to subscribe/unsubscribe a plugin to/from a market |
| ![feature] | <details><summary>Plugin persistence via NBT save/load</summary><br>Plugin instances, execution order, market subscriptions, custom settings, and plugin-specific data survive server restarts. Fresh worlds fall back to default plugin creation.</details> |
| ![feature] | `PluginSyncData` — network-transferable plugin state bundle with `StreamCodec`, now includes custom settings |
| ![improvement] | `PluginSyncData.fromServerPlugin()` — factory method to build sync data from a server plugin instance |
| ![improvement] | `ServerPluginManager` uses `LinkedHashMap` for deterministic plugin execution order |
| ![bugfix] | `GenericPluginData` private constructor not assigning `enabled` field — deserialized plugins always showed as disabled |
| ![bugfix] | `AsyncMarketManager.createMarketAsync()` sent `FunctionType.MarketExists` instead of `FunctionType.CreateMarket` |
| ![bugfix] | `AsyncMarketManager.deleteMarketAsync()` sent `FunctionType.MarketExists` instead of `FunctionType.DeleteMarket` |
| ![hardening] | Plugin registration split into server-safe static init and client-only `clientSetup()` to prevent dedicated server crash from loading GUI classes |
