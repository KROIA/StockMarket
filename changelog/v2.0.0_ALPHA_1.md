# v2.0.0_ALPHA_1

Plugin system architecture, comprehensive bug fixes, and security hardening.

## Security

| Tag | Change |
|---:|---|
| ![bugfix] | <details><summary>Integer overflow exploit in `CreateOrderRequest` allowed orders without sufficient funds</summary><br>Volume × price calculation could overflow to negative, passing the balance check. Now uses `Math.multiplyExact()` and returns `INVALID_VOLUME` on overflow.</details> |
| ![bugfix] | <details><summary>Broken executor-player filter in `ActiveOrdersRequest.passesFilter()`</summary><br>The filter compared against the requester's UUID instead of the order's executor UUID, exposing other players' orders.</details> |
| ![bugfix] | <details><summary>Permission bypass in `ActiveOrdersRequest`</summary><br>Non-admin players could view all orders. Now restricted to orders for bank accounts the player is a member of.</details> |
| ![bugfix] | <details><summary>`RuntimeException` crash for `INTER_MARKET` order type in `CreateOrderRequest`</summary><br>Now returns `NO_SUCH_MARKET` before any funds are locked, preventing both the crash and a fund-locking leak.</details> |
| ![hardening] | <details><summary>SQL injection vector in database filter system</summary><br>All filters (`EqualityFilter`, `DateFilter`, `UUIDFilter`) converted from string concatenation to parameterized `PreparedStatement` queries.</details> |

## Internal

| Tag | Change |
|---:|---|
| ![bugfix] | <details><summary>Division by zero in `Order.getAverageExecutionPrice()` with unfilled orders</summary><br>Also resolves the same exception path triggered via `InterMarketOrder.getHistoricalRecord()`.</details> |
| ![bugfix] | <details><summary>`ConcurrentModificationException` in `MatchingEngine` during order matching</summary><br>Iteration now uses snapshot copies with deferred removal.</details> |
| ![bugfix] | <details><summary>Null-pointer crash in `onClientTickEvent` when leaving a server</summary><br>`CLIENT_INSTANCES` could be null after disconnect.</details> |
| ![bugfix] | <details><summary>Unsafe `long`-to-`int` casts in `DynamicIndexedArray.moveOffset()` and `getArrayIndex()`</summary><br>Could cause `ArrayIndexOutOfBoundsException` or silent data corruption when index offset exceeds int range.</details> |
| ![bugfix] | <details><summary>Integer overflow in `MatchingEngine` volume/price calculations</summary><br>Now uses `Math.multiplyExact()`; overflowing orders are safely canceled instead of producing negative costs.</details> |
| ![bugfix] | <details><summary>Incorrect `PriorityQueue` iteration in `Orderbook`</summary><br>`getRawVolume`, `getRawVolumeRounded`, `getPriceWhenConsumingVolume`, and `getCapital` now correctly scan all orders instead of assuming sorted iteration order.</details> |
| ![bugfix] | <details><summary>Switch fallthrough and missing operator dispatch in `VirtualOrderBookCache.ManipulationData.apply()`</summary><br>RANGE/ARRAY cases fell through; ADD operator always called `set` instead of `add`. Added `break` statements and correct operator dispatch.</details> |
| ![bugfix] | <details><summary>Division by zero in `PID.update()` when dt is zero</summary><br>Two updates in the same millisecond produced NaN via the derivative term, permanently corrupting the controller. Derivative term now returns 0 when dt=0.</details> |
| ![bugfix] | <details><summary>`ServerMarket.load()` not restoring candle state</summary><br>First candle after restart showed a false price spike from uninitialized candle fields. Now initializes open/high/low/startTime from the loaded price.</details> |
| ![bugfix] | <details><summary>`AsyncMarket.setSettingsAsync()` omitting settings from request</summary><br>The `settings` parameter was never passed to `InputData.of()`, causing NPE on the server side. Feature was completely broken.</details> |
| ![bugfix] | <details><summary>Volume jump on first tick of `DefaultOrderbookVolumeDistributionPlugin`</summary><br>`RuntimeData.lastMillis` defaulted to 0, producing an enormous deltaT on first update. Now initialized to current time in constructor.</details> |
| ![bugfix] | <details><summary>`Orderbook.getRawVolume()` undercounting at exact market price</summary><br>Used `peek()` on a single queue instead of iterating all orders in both buy and sell queues at market price.</details> |
| ![bugfix] | <details><summary>`ServerMarketManager.load()` discarding all markets on single failure</summary><br>One corrupted market prevented all successfully-loaded markets from being applied. Now always applies loaded markets and logs individual failures.</details> |
| ![bugfix] | <details><summary>`DataManager.savePriceCandlesToSQL()` lock not released on exception</summary><br>Used `thenRun` which skips on failure; replaced with `whenComplete` to release the save lock on both success and failure paths.</details> |
| ![bugfix] | `PriceHistoryData.getMinPrice()`/`getMaxPrice()` used `Integer.MAX_VALUE`/`MIN_VALUE` as sentinels for `long` fields — incorrect for extreme prices |
| ![bugfix] | <details><summary>`DynamicIndexedArray.set(float[])` and `add(float[])` wrong source offset</summary><br>When the virtual start index was before the array range, `srcOffset` was not calculated before clamping, placing data at wrong positions.</details> |
| ![bugfix] | Null-pointer crash in `TargetPriceBot.update()` when market data entry is missing |
| ![bugfix] | Null-pointer crash in `DefaultOrderbookVolumeDistributionPlugin.onMarketUnsubscribed()` for improperly subscribed markets |
| ![bugfix] | <details><summary>`DatabaseManager` static singleton caused stale connections on reload</summary><br>Converted to instance-based with proper lifecycle; connections created on server start and closed on server stop.</details> |
| ![bugfix] | <details><summary>`PreparedStatement` and `ResultSet` resource leaks</summary><br>`MarketPriceManager` and `OrderRecordManager` now use try-with-resources.</details> |
| ![bugfix] | Null guard for database row mapping in `MarketPriceManager` and `OrderRecordManager` |
| ![bugfix] | Error logging for swallowed `SQLException` in `OrderRecordManager.queueRecord()` |
| ![bugfix] | Bounds check for `Type` enum ordinal during `Order` deserialization |
| ![bugfix] | Misleading success logging in `DatabaseManager` — now only logs on actual success |
| ![bugfix] | Null-safety check for default volume provider in `ServerMarket` |
| ![bugfix] | `MarketPriceHistoryRequest` completing future with null — now returns empty data |
| ![bugfix] | Typos in `ServerMarket` method names (`getIncomingSelMarketOrders` → `getIncomingSellMarketOrders`) |
| ![bugfix] | Boolean logic inversion in `DataManager.loadSettings()` |
| ![bugfix] | `MatchingEngine` log messages referenced non-existent `getOrderID()` method |
| ![bugfix] | NPE in `/stockmarket testdb` async callbacks when player disconnects or command run from RCON |
| ![improvement] | Deduplicated `roundConservative()` — `VirtualOrderbook` now delegates to `DynamicIndexedArray` |
| ![improvement] | Standardized error logging in all `load()` methods — `PID`, `NormalizedRandomPriceGenerator`, `User` now log before returning false |
| ![improvement] | Candle save interval now dynamically configurable at runtime |
| ![improvement] | `DatabaseManager.commitTransaction()` returns boolean success indicator |
| ![improvement] | Warning log for timeout in `MatchingEngine` virtual volume draining |
