## Overview

Major architectural update introducing the plugin system for extensibility and improved management interfaces.

---

## Features

### Plugin System
- Added complete plugin architecture with async/sync support
- Implemented plugin API with client and server interfaces
- Created plugin manager for lifecycle management
- Added plugin-market interaction layer
- Implemented plugin cache system for performance

### Built-in Plugins
- **TargetPriceBot**: Automated market maker with price targeting
- **VolatilityPlugin**: Market volatility simulation
- **DefaultOrderbookVolumeDistributionPlugin**: Volume distribution calculator

### UI Improvements
- Enhanced Management UI
- Plugin configuration interface
- Improved market management screens

### Unit Test Framework
- Integrated ModUtilities test suite framework
- Added `/stockmarket test` command with master/slave server awareness
- 14 test categories (7 run on both server types, 7 master-only)
- 5 test suites with 35 tests: Order, DynamicIndexedArray, PID, RandomWalk, MarketIntegration
- Migrated existing startup tests into the new framework and removed old `Testing.java`

## Bug Fixes

### Critical
- Fixed division by zero in `Order.getAverageExecutionPrice()` when an order has no filled volume; also resolves the same exception path triggered via `InterMarketOrder.getHistoricalRecord()`
- Fixed integer overflow exploit in `CreateOrderRequest` that allowed players to place orders without sufficient funds; volume × price now uses `Math.multiplyExact()` and returns `INVALID_VOLUME` on overflow
- Fixed `ConcurrentModificationException` in `MatchingEngine` during order matching — iteration now uses snapshot copies with deferred removal

### Security
- Fixed broken executor-player filter in `ActiveOrdersRequest.passesFilter()` — the filter now correctly compares against the order's executor UUID instead of the requester
- Fixed permission bypass in `ActiveOrdersRequest` — non-admin players can now only view orders for bank accounts they are members of

### High
- Fixed null-pointer crash in `StockMarketModBackend.onClientTickEvent()` when leaving a server (`CLIENT_INSTANCES` could be null)
- Added null guard for database row mapping in `MarketPriceManager` and `OrderRecordManager` to prevent null entries from being added to result lists
- Added error logging for swallowed `SQLException` in `OrderRecordManager.queueRecord()` so failed history writes are no longer silent
- Fixed `RuntimeException` crash for `Order.Type.INTER_MARKET` in `CreateOrderRequest`; the request now returns `NO_SUCH_MARKET` before any funds are locked
- Persist userMap (admin/op grants) across server restarts in `ServerMarketManager` save/load

### Medium
- Added bounds check for the `Type` enum ordinal during `Order` deserialization to prevent corrupted save data from crashing the server
- Fixed misleading success logging in `DatabaseManager` — "Successfully connected" is now only logged when table creation also succeeds
- Fixed incorrect `PriorityQueue` iteration in `Orderbook` — `getRawVolume`, `getRawVolumeRounded`, `getPriceWhenConsumingVolume`, and `getCapital` now correctly scan all orders instead of assuming sorted iteration order
- Added null-safety check for default volume provider in `ServerMarket`
- Removed unused states (`EXEC_SAVE`, `EXEC_LOAD`, `EXEC_ENCODING`) from `ServerPluginManager` `State` enum
- Added warning log for timeout in `MatchingEngine` virtual volume draining
- Fixed `ManagementScreen` not unsubscribing from market price stream on close (stream leak)
- Fixed `DevTestScreen` not unsubscribing from market price stream on close (stream leak)
- Fixed `TradeScreen` null pointer crash when the market disappears while trading
- Fixed `PreparedStatement` and `ResultSet` resource leaks in `MarketPriceManager` and `OrderRecordManager` (now uses try-with-resources)
- Fixed `MarketPriceHistoryRequest` completing future with null — now returns empty data instead
- Made candle save interval dynamically configurable at runtime (reads from settings on each update)
- Improved transaction failure handling in `DatabaseManager.commitTransaction()` — now returns boolean success indicator

### Low
- Fixed typos in `ServerMarket` method names (`getIncomingSelMarketOrders` → `getIncomingSellMarketOrders`, etc.)
- Fixed boolean logic inversion in `DataManager.loadSettings()`
- Fixed `/stockmarket db <table> count` command to correctly route to the specified table manager

## API Changes

### Breaking Changes
- Introduction of plugin API may affect custom implementations
- Market interface extended with plugin hooks

### New APIs
- `IAsyncPlugin`, `IServerPlugin`, `IClientPlugin`, `ISyncServerPlugin`
- `IPluginManager` and implementations
- `IPluginMarket`, `IPluginOrderBook` interaction interfaces
- `IVolumeDistributionCalculator` for custom volume calculations

- Latent SQL injection vector in `EqualityFilter`

**Low:**
- Static mutable `timeOffsetMS` shared across all `ClientMarket` instances
