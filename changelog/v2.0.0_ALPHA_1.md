# v2.0.0_ALPHA_1 — In Development

Plugin system architecture, comprehensive bug fixes, and security hardening.

## Security

| Tag | Change |
|---:|---|
| ![bugfix] | <details><summary>Integer overflow exploit in `CreateOrderRequest` allowed orders without sufficient funds</summary><br>Volume × price calculation could overflow to negative, passing the balance check. Now uses `Math.multiplyExact()` and returns `INVALID_VOLUME` on overflow.</details> |
| ![bugfix] | <details><summary>Broken executor-player filter in `ActiveOrdersRequest.passesFilter()`</summary><br>The filter compared against the requester's UUID instead of the order's executor UUID, exposing other players' orders.</details> |
| ![bugfix] | <details><summary>Permission bypass in `ActiveOrdersRequest`</summary><br>Non-admin players could view all orders. Now restricted to orders for bank accounts the player is a member of.</details> |
| ![bugfix] | <details><summary>`RuntimeException` crash for `INTER_MARKET` order type in `CreateOrderRequest`</summary><br>Now returns `NO_SUCH_MARKET` before any funds are locked, preventing both the crash and a fund-locking leak.</details> |
| ![hardening] | <details><summary>SQL injection vector in database filter system</summary><br>All filters (`EqualityFilter`, `DateFilter`, `UUIDFilter`) converted from string concatenation to parameterized `PreparedStatement` queries.</details> |

## Internal

| Tag | Change |
|---:|---|
| ![bugfix] | <details><summary>Division by zero in `Order.getAverageExecutionPrice()` with unfilled orders</summary><br>Also resolves the same exception path triggered via `InterMarketOrder.getHistoricalRecord()`.</details> |
| ![bugfix] | <details><summary>`ConcurrentModificationException` in `MatchingEngine` during order matching</summary><br>Iteration now uses snapshot copies with deferred removal.</details> |
| ![bugfix] | <details><summary>Null-pointer crash in `onClientTickEvent` when leaving a server</summary><br>`CLIENT_INSTANCES` could be null after disconnect.</details> |
| ![bugfix] | <details><summary>Unsafe `long`-to-`int` casts in `DynamicIndexedArray.moveOffset()` and `getArrayIndex()`</summary><br>Could cause `ArrayIndexOutOfBoundsException` or silent data corruption when index offset exceeds int range.</details> |
| ![bugfix] | <details><summary>Integer overflow in `MatchingEngine` volume/price calculations</summary><br>Now uses `Math.multiplyExact()`; overflowing orders are safely canceled instead of producing negative costs.</details> |
| ![bugfix] | <details><summary>Incorrect `PriorityQueue` iteration in `Orderbook`</summary><br>`getRawVolume`, `getRawVolumeRounded`, `getPriceWhenConsumingVolume`, and `getCapital` now correctly scan all orders instead of assuming sorted iteration order.</details> |
