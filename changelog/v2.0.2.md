# v2.0.2 — In Development

## UI / UX

| Tag | Change |
|---:|---|
| ![bugfix] | <details><summary>TradeScreen crash on non-`.` decimal locales and non-finite prices</summary><br>`FavoritesBar.formatPriceWithSeparators` called `String.format("%.2f", price)` (default locale) and parsed the result with `indexOf('.')` + `substring`. On any locale where the decimal separator is not `.` (e.g. de_DE, fr_FR), or for `NaN`/`Infinity` inputs, `indexOf` returned `-1` and `substring(0, -1)` threw `StringIndexOutOfBoundsException`, crashing the screen.<br><br>**Fix:** pin the formatter to `Locale.ROOT`, short-circuit on `!Double.isFinite(price)`, and add a defensive `dot < 0` fallback.</details> |
| ![bugfix] | <details><summary>Numeric formatting now uses `.` as decimal separator on all locales</summary><br>Across the UI, `String.format("%.Nf", value)` used the JVM default locale, showing `1,23` instead of `1.23` on `,`-decimal locales (de_DE, fr_FR, ru_RU, etc.). `CandlestickChart`'s `decimals > 3` parsing branch additionally produced garbage like `Inf'ini'ty` for non-finite inputs.<br><br>**Fix:** pinned ~29 numeric format calls across 12 UI files to `Locale.ROOT`, and added a `dot < 0` fallback in `CandlestickChart.formatCompact` for the parsing branch. Display dates (`SimpleDateFormat`) intentionally remain locale-dependent.</details> |
