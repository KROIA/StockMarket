# v2.0.3 — In Development

## CLI

| Tag | Change |
|---:|---|
| ![feature] | <details><summary>New admin command `/stockmarket <market> remove`</summary><br>Deletes a market without opening the Management GUI (permission level 2). `<market>` accepts the item's registry name in quotes (e.g. `"minecraft:iron_ingot"`) or a numeric market ID. When several markets share a registry name (items differing only in components), the command lists the matching `name=id` candidates so the right one can be removed by number — this also makes broken markets addressable that no longer resolve to an item. Tab completion suggests all known markets. Uses the existing deletion request, so it is safe on master/slave setups.</details> |

## UI / UX

| Tag | Change |
|---:|---|
| ![bugfix] | <details><summary>Money-market limit orders no longer drawn on Item/Item pair charts</summary><br>With active limit orders on an Item/Money market, switching the trade screen to Item/Item pair mode drew those money-denominated orders on the cross-rate chart at nonsensical prices.<br><br>**Fix:** the order-marker overlay now branches on pair mode first — in pair mode only inter-market orders are rendered — and the trade screen clears/restores the overlay's market binding on mode switches so stale market IDs cannot leak into pair mode. Money-mode rendering is unchanged.</details> |
| ![bugfix] | <details><summary>Markets with drifted or unresolvable items are selectable and removable again</summary><br>Markets whose item components changed over time (e.g. items with volatile data components) could become unselectable in the Management screen, and markets whose item can no longer be resolved at all rendered blank and could never be removed via the GUI.<br><br>**Fix:** market selection now passes the market ID directly instead of round-tripping through an ItemStack, so component drift no longer breaks selection. Unresolvable markets render as a barrier-icon "broken market" placeholder with an explanatory tooltip, stay selectable, and can be removed with the regular Remove button.</details> |
| ![bugfix] | <details><summary>Deleted markets disappear from all clients immediately</summary><br>Removing a market (via command or GUI) left it selectable in every connected client's trade screen with a dead chart and no price updates, because clients were never told about deletions and their market cache only ever grew.<br><br>**Fix:** the server now broadcasts market removals to all players (relayed across master/slave setups). Clients purge the cached market, stop its price stream, and evict cross-rate caches; open screens deselect the market gracefully in both money and pair mode and rebuild the favorites bar. The client market list additionally prunes stale entries on refresh, and the server-side price stream shuts itself down when its market is gone instead of looping silently.</details> |

## Internal

| Tag | Change |
|---:|---|
| ![hardening] | Market preset lookup by market ID is now component-aware: it first matches the preset's exact registered item identity and only falls back to the registry name, so items differing only in components resolve to the correct preset. |
